## What this does

Exportiert `createMcpHttpHandler(opts: { buildServer: () => Server | Promise<Server> }): (req: Request) => Promise<Response>`.

Implementierung (Pseudocode):

```ts
import { WebStandardStreamableHTTPServerTransport }
  from "@modelcontextprotocol/sdk/server/webStandardStreamableHttp.js";
import { Server } from "@modelcontextprotocol/sdk/server/index.js";

export interface CreateMcpHttpHandlerOptions {
  buildServer: () => Server | Promise<Server>;
}

export function createMcpHttpHandler(opts: CreateMcpHttpHandlerOptions) {
  return async (req: Request): Promise<Response> => {
    const transport = new WebStandardStreamableHTTPServerTransport({
      sessionIdGenerator: undefined, // stateless mode -- spike scope
    });
    const server = await opts.buildServer();
    await server.connect(transport);
    return transport.handleRequest(req);
  };
}
```

**Rationale für die Form:**
- `buildServer` ist factory (nicht Server-Instance) damit jeder Request in stateless mode eine frische Instance bekommt -- keine Cross-Request-Leaks.
- `Request → Response` Web-Standard-API passt 1:1 in Next.js 15 App-Router-Route-Handler (`export async function POST(req: Request) { return handler(req); }`).
- Web-Standard-Variante (`webStandardStreamableHttp.js`) statt Node-Wrapper (`streamableHttp.js`) weil Next.js Web `Request`/`Response` nutzt, nicht `IncomingMessage`/`ServerResponse`.
- Stateless mode für Spike: keine Session-State-Management-Komplexität in T01. Stateful mode kann später in S02 für SSE-Streaming nachgerüstet werden.

**Was T01 explizit NICHT macht:**
- Keine Auth (kommt in T02)
- Keine Tools registriert (Stub-Tool nur im Test)
- Keine Origin-Validation (kommt in S03)
- Keine Routes (`src/app/api/mcp/route.ts` ist T02)

**Test (`http-transport.test.ts`):**
```ts
import { describe, it, expect } from "vitest";
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { ListToolsRequestSchema, CallToolRequestSchema }
  from "@modelcontextprotocol/sdk/types.js";
import { createMcpHttpHandler } from "./http-transport";

function buildStubServer() {
  const server = new Server(
    { name: "stub", version: "0.0.1" },
    { capabilities: { tools: {} } }
  );
  server.setRequestHandler(ListToolsRequestSchema, async () => ({
    tools: [{ name: "ping", description: "stub", inputSchema: { type: "object" } }],
  }));
  server.setRequestHandler(CallToolRequestSchema, async () => ({
    content: [{ type: "text", text: "pong" }],
  }));
  return server;
}

describe("createMcpHttpHandler", () => {
  it("responds to initialize + tools/list + tools/call roundtrip", async () => {
    const handler = createMcpHttpHandler({ buildServer: buildStubServer });

    // initialize
    const initRes = await handler(new Request("http://test/mcp", {
      method: "POST",
      headers: { "content-type": "application/json", accept: "application/json, text/event-stream" },
      body: JSON.stringify({
        jsonrpc: "2.0", id: 1, method: "initialize",
        params: { protocolVersion: "2025-06-18", capabilities: {}, clientInfo: { name: "test", version: "0" } },
      }),
    }));
    expect(initRes.status).toBe(200);

    // tools/list
    const listRes = await handler(new Request("http://test/mcp", {
      method: "POST",
      headers: { "content-type": "application/json", accept: "application/json, text/event-stream" },
      body: JSON.stringify({ jsonrpc: "2.0", id: 2, method: "tools/list" }),
    }));
    expect(listRes.status).toBe(200);
    // Parse response — could be SSE or JSON depending on transport choice
    const body = await listRes.text();
    expect(body).toContain("ping");
  });
});
```

Wenn die Response SSE-formatted ist (text/event-stream), parsen wir den `data:`-line. Falls JSON: `JSON.parse`. Der Test ist tolerant (`toContain("ping")`).
