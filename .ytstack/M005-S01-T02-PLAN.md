## What this does

`createJamSession(userId, {name?, budgetTotal?})`: resolves tier from
`prisma.subscription` (api-keys route pattern), `canUseFeature("jamSessions")`
else `Err.forbidden`; validates budget int 1..100 (default 30); caps open
sessions per host at 3 (`Err.limitReached`); creates the playlist (default
name "Jam Session <date>", stripHtml on custom names — crud.ts pattern) and
the `JamSession` row in one transaction; returns the session incl. shareToken.

`closeJamSession(sessionId, userId)`: ownership via `findFirst {id,
hostUserId}` → `Err.notFound`; idempotent (already-closed returns success);
sets status closed + closedAt.

`vetoJamEntry(sessionId, entryId, userId)`: session ownership → notFound;
entry must belong to session → notFound; only `pending` entries can be vetoed
(`Err.conflict`); sets entry status "vetoed" (T05's completion hook skips
vetoed entries; an already-running generation is not cancelled — budget stays
consumed, accepted per pitch).

Routes are thin `authRoute` wrappers with zod bodies mapping to
`resultResponse` (201 on create).
