## What this does

Pending cards get an X (aria-label "Remove request") calling
`DELETE /api/jam-sessions/[id]/entries/[entryId]`; on success the state
refreshes immediately (card disappears for host AND guests — vetoed is
filtered server-side). An "End session" button (inline confirm, no window
dialog) POSTs `.../close`; the header then shows "Session ended" and the
controls disable. Budget countdown already ships (T01). Every failure path
lands in a toast (no silent branches).
