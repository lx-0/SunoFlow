# M005-S01-T01 -- Task Plan

**Summary:** Prisma schema + migration: `JamSession` model (host, linked playlist, share token, budget total/used, status open/closed) plus `JamSessionEntry` linkage tying generated songs to the session with prompt text + guest display name; migration applies cleanly on a throwaway DB.

## Files

- Modify: `prisma/schema.prisma` (add `JamSession` + `JamSessionEntry` models; backrefs on `User`, `Playlist`, `Song`)
- Create: `prisma/migrations/<timestamp>_add_jam_sessions/migration.sql` (generated by `prisma migrate dev`)

## What this does

Two new models following house conventions (cuid ids, String status with
comment enums, `@unique @default(cuid())` share token like
`PlaylistCollaborator.inviteToken`):

```prisma
model JamSession {
  id          String    @id @default(cuid())
  hostUserId  String
  playlistId  String    @unique          // 1:1 — the session IS a playlist view
  shareToken  String    @unique @default(cuid())
  status      String    @default("open") // open, closed
  budgetTotal Int       @default(30)     // max generations for this party
  budgetUsed  Int       @default(0)
  createdAt   DateTime  @default(now())
  closedAt    DateTime?

  host     User              @relation("JamSessionHost", fields: [hostUserId], references: [id], onDelete: Cascade)
  playlist Playlist          @relation(fields: [playlistId], references: [id], onDelete: Cascade)
  entries  JamSessionEntry[]

  @@index([hostUserId, status])
}

model JamSessionEntry {
  id         String   @id @default(cuid())
  sessionId  String
  songId     String?  @unique            // set when generation starts; SetNull if song deleted
  promptText String
  guestName  String?
  guestKey   String                      // anonymous per-device guest id (rate limiting)
  status     String   @default("pending") // pending, ready, failed, vetoed
  createdAt  DateTime @default(now())

  session JamSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  song    Song?      @relation(fields: [songId], references: [id], onDelete: SetNull)

  @@index([sessionId, createdAt])
  @@index([sessionId, guestKey])
}
```

Backrefs: `User.jamSessions JamSession[] @relation("JamSessionHost")`,
`Playlist.jamSession JamSession?`, `Song.jamEntry JamSessionEntry?`.
Entry.status is denormalized from the song's generationStatus on purpose —
it also carries `vetoed`, which has no song-side equivalent, and survives
song deletion (`SetNull`) for the post-party recap.
