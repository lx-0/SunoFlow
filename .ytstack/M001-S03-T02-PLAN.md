## What this does

**Pass 1 -- Goals klar setzen:**
- Reduce mental-model overload (30 useState mental-trackable → 5 primary cells).
- Persona-First Workflow als happy path.
- Mobile-default Layout (per §7.5 Persona).
- 0 Feature-Verlust (Sub-Flows bleiben erreichbar).
- Compatible with D10 (Tabs: simple/advanced/mashup/compare).

**Pass 2 -- Disclosure-Levels definieren:**

Level 0 -- **Default Simple Mode** (entspricht D10 simple-Tab):
- 5 Primary-Felder (Persona-Picker, Style, Lyrics, Instrumental-Toggle, Title)
- 1 Submit-Button
- (kein Custom-Mode-Toggle sichtbar -- Default ist Persona-First)

Level 1 -- **Advanced Toggle expanded** (innerhalb simple-Tab):
- Custom-Mode-Toggle (an = User schreibt Lyrics selbst)
- Style-Boost-Button neben Style-Feld
- Preset-Picker (read-only: aus /authoring waehlen, hier nicht editieren)
- Template-Picker (read-only)

Level 2 -- **Separate Tabs** (D10: advanced/mashup/compare):
- Tab "Advanced" = Power-User-Mode mit zusaetzlichen Knoepfen, alles inline sichtbar (kein Disclosure-Toggle mehr)
- Tab "Mashup" = `/mashup` Workflow (eigene Logik)
- Tab "Compare" = `/compare` Workflow

Level 3 -- **Auto-Mode** (eigener Tab oder /generate/auto):
- Single high-level prompt, LLM macht den Rest
- Maximal entlastet, fuer Lazy-Power-User

Plus Sub-Flow-Triggers:
- "Need lyrics? Generate them →" link unter Lyrics-Feld (oeffnet sub-flow ggf. als BottomSheet)
- Style-Boost-Button laeuft inline (LLM-Call ueberschreibt Style-Feld)

**Pass 4 -- Behavior Notes:**

- **Persona-Pick:** Auto-fill Style + Lyrics + InstrumentalDefault. User kann ueberschreiben.
- **Custom-Mode-Toggle:** Lyrics-Feld bleibt sichtbar; nur Auto-Fill aus Persona/Preset wird disabled.
- **Style-Boost:** Disabled wenn Style-Feld leer. Loading-State 1-2s.
- **Submit:** Disabled if Style empty AND Lyrics empty AND no Persona selected.
- **Credit-low:** Below 10% remaining shows banner above submit. Below 0: button laesst UpgradeModal aufpoppen.
- **Tabs persisted via `?tab=` URL param** (per D10).

**Pass 5 -- Desktop Variations:**

Desktop (md: 768px+) kann same Layout horizontal anordnen oder Two-Column-Layout (Persona/Style links, Lyrics/Title rechts). ASCII zeigt Two-Column-Variante.

**Pass 6 -- Implementation Hints fuer M002:**

- GenerateForm.tsx wird refactored, nicht ersetzt. (Hot-File, §7.6 Constraint.)
- Sub-App-Auswanderung: Preset-CRUD + Template-CRUD wandern in /authoring (per D11). Hier nur Read-Only-Picker.
- useState-Reduktion-Target: 30 → ~12 (siehe Inventory §4 §6).
- Naming-Drift-Decision (stylePrompt vs tags vs style; lyrics vs prompt) muss VOR Refactor -- DECISIONS-Entry D15 noetig.
