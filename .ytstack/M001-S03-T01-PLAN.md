## What this does

**Pass 1 -- Parameter-Enumeration:**
- `grep -nE 'useState' src/components/GenerateForm.tsx` -- jedes State-Cell ist ein Parameter-Kandidat.
- `grep -nE 'const \[' src/components/GenerateForm.tsx` -- destructured state.
- `cat src/components/generate-form/types.ts` -- formal Param-types.
- `cat src/lib/generation/request.ts` -- Zod schema fuer Backend-Validation.
- Result: Liste mit ~30 Parameter-Kandidaten.

**Pass 2 -- Pro Parameter erfassen:**
| Param | Type | Default | Pflicht? | Persona-fillable? | UI-Widget | Note |

Spalten ergaenzen: 
- **Persona-fillable** -- Wenn User eine Persona auswaehlt, was wird auto-gesetzt? Aus `useGenerateFormData.ts` + `autoFillGenerationFields` in `generate-form/api.ts`.
- **Preset-fillable** -- Selbe Frage fuer Presets.

**Pass 3 -- Klassifizierung Primary / Secondary / Advanced:**
- **Primary** = Anfaenger-Friendly, jeder neue User stellt das ein (prompt, persona oder preset, style).
- **Secondary** = nicht-Anfaenger, aber Power-User stellen das regelmaessig (title, model variant, tags).
- **Advanced** = selten direkt-editiert, Persona-Default deckt das (custom-lyrics, weirdness, structured, voice variant, tempo range).

Begruendung pro Klassifizierung: 1 Satz "warum hier".

**Pass 4 -- Friction-Cross-Check:**
- Aus FRICTION-AUDIT: 30 useState, 11 native widgets (5 input, 3 select, 3 textarea).
- Pruefen: hat jeder useState einen Param-Eintrag? Wenn nicht: was sind die Extra-useState (UI-State wie showAdvanced, isLoading, error)?

**Pass 5 -- Persona/Preset/Template Override-Mapping:**
- Wie viele Parameter werden durch Persona-Selektion ueberschrieben vs. behalten?
- Selbe Frage fuer Preset und PromptTemplate.
- Tabelle: Param-x-Override-Source.

Output-Datei-Struktur:
```
## §1 Parameter Catalog (alle ~30)
## §2 Classification Primary/Secondary/Advanced (mit Begruendung)
## §3 Persona/Preset/Template Override-Mapping
## §4 UI-State vs Domain-State (was ist Param, was ist Form-UI-State)
## §5 Backend-Persistence-Map (welche Params wandern in Song-Model)
## §6 Friction-Cross-Check vs FRICTION-AUDIT §1
```
