## What this does

`getJamSessionState(shareToken)` returns a guest-safe payload:
- `session`: status (guests see "closed" as party-over), name (playlist
  name), hostName, budgetTotal/budgetUsed (countdown).
- `entries`: all non-vetoed entries ordered by createdAt asc — vetoed cards
  disappear for guests; failed ones stay visible as honest feedback. Each
  entry: id, status, promptText, guestName, createdAt + a LEAN song card
  (id, title, imageUrl, duration, generationStatus) — no sunoJobId or other
  internals leak through the token surface.
- `nowPlaying`: best-effort from the host's PlaybackState (songId + position,
  only when that song is a member of the session playlist) — may lag behind
  real playback; S02's host page owns improving this if needed.

Route `GET /api/jam/[token]`: anonRoute with a deliberately GENEROUS IP rate
limit (600/min) — an entire party polls from ONE NAT'd Wi-Fi IP (15 guests ×
12 polls/min ≈ 180/min legit traffic); the limit is an abuse backstop, not a
throttle. Token is the auth: unknown token → 404 via resultResponse.
