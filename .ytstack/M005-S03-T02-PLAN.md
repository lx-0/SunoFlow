## What this does

A fixed bottom composer (input + send) with vibe chips above it that seed
the text. `guestKey` is minted once per device (localStorage). Submit POSTs
the tokened prompt endpoint; on 201 the RETURNED entry card is inserted
into the list immediately (true server card, not a fake), the field clears.
Every error path is visible: server messages (rate limit "wait until one
finishes", budget "used up", closed "party has ended") render inline under
the composer with aria-live (no Toast provider exists on the public page).
When the session is closed/expired or the budget hits 0, the composer is
replaced by the corresponding state text.
