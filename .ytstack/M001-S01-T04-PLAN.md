## What this does

**Pro Surface (5 Metriken-Familien):**

1. **Lines of Code (LOC)** -- `wc -l` als Groessen-Proxy.
2. **State surface** -- Anzahl `useState` / `useReducer` / `useRef` + custom Hooks im File. Quantifiziert "wie viele Dinge gleichzeitig im Kopf des Components".
3. **Props surface** -- Anzahl Props (aus Props-Interface oder Function-Signature). Quantifiziert externe Coupling.
4. **Conditional rendering branches** -- Anzahl `&&` / `?:` / `if (...)` in JSX-naher Region. Proxy fuer "hidden complexity".
5. **Imports / Dependencies** -- Anzahl `import`-Lines + zaehl explizit: wie viele Modals, wie viele Icons, wie viele Sub-Components.

**Surface-spezifisch:**

- **GenerateForm**: Anzahl Form-Fields (Inputs + Selects + Textareas + Checkboxes), Anzahl Tabs/Modes (Simple vs Advanced), Anzahl Persona/Preset-Side-Effects, Anzahl async Operations (style-boost, lyrics-generate, etc.).
- **LibraryView**: Anzahl Filter-Achsen, Anzahl Sort-Optionen, Anzahl Bulk-Operations, View-Modes (grid/list/?), Selection-States.
- **SongDetailView**: Anzahl Action-Buttons im SongActionsBar (gekoppelt), Anzahl Modals (Cover-Art, Embed, Remix, Variation, SeparateVocals, Report), Anzahl conditional CTAs nach Song-State.
- **AppShell**: Top-Nav-Items (17 bestaetigt aus T01), Auth-State-Branches (logged-out/logged-in/admin), GlobalPlayer-Slot-Wiring.
- **GlobalPlayer**: Anzahl Race-Guards / Token-Patterns (load-generation, peaks-worker async), Anzahl Player-Events.
- **QueueContext**: Anzahl Context-Werte exponiert, Anzahl Operations (`addItem`, `updateItem` (dual-signature), shuffle/repeat/seek/...).

**Methode:** Verwende `grep -c` / `awk` patterns gegen die Files. Keine semantische Bewertung -- nur Zahlen. Interpretation/Empfehlungen kommen in S02/S03.
