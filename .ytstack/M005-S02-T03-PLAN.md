## What this does

The host console keeps a ref map of entry→status across polls. When an
entry transitions pending→ready DURING this page's lifetime, the full song
row is fetched (`GET /api/songs/[id]`, host owns it) and appended to the
running play queue via `useQueue().addToQueue(songToQueueSong(song))`, with
a toast. Entries that are ALREADY ready on first load are seeded as known
and NOT enqueued (a host reopening the console mid-party must not re-append
half the playlist). Dedup ref prevents double-appends across poll jitter.
