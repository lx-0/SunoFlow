## What this does

Host GET surface: `listJamSessions(userId)` (newest first) and
`getJamSession(sessionId, userId)` (summary + playlist name; ownership →
404). Entry point on /playlists: a "Jam" button (PartyPopper), rendered only
for `session.user.subscriptionTier === "studio"`, opens an inline form
(optional name + budget 1..100 default 30) mirroring the existing create-form
pattern; success routes to `/party/[id]`. The host console at `/party/[id]`
(deliberately NOT under `/jam/` — that namespace is the public guest surface
in S03) shows session status, budget countdown, the share URL
(`/jam/<token>`, QR overlay lands in T02), and the live queue by polling the
same guest state endpoint the phones use (dogfoods the token surface,
including the middleware pass-through).
